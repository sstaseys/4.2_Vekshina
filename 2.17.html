Описание системы

Система электронной регистратуры — это информационная система, предназначенная для автоматизации процессов записи пациентов на приём к врачам, мониторинга и управления потоками пациентов, формирования расписания приёма специалистов. Она обеспечивает информационную поддержку медицинских специалистов в предоставлении сведений о пациентах, оказываемой им помощи и деятельности медицинского учреждения в целом.

 Система в новой версии должна включать кабинет врача-общей практики, в котором будет обеспечено ведение медицинской документации. Врач общей практики сможет заполнять медицинскую документацию пациентов, включая истории болезни, результаты диагностических исследований, рецепты на лекарства и направления к специалистам.

 Функции системы с точки зрения врача поликлиники, который пользуется внутренней системой (вариант 3):

1.            Работать с расписанием;

2.            Выписывать направления и записывать пациента;

3.            Вести прием пациентов по списку записанных;

4.            Контролировать очередь по профилю;

5.            Работать с Электронной медицинской картой пациента;

6.            Работать с картами диспансерных больных;

7.            Просматривать журналы госпитализации, вызовы СМП, направлений;

8.            Просматривать журналы вакцинации;

9.            Просматривать справочник вакцин и справочник «Национальный календарь» профилактических прививок»;

10.         Работать с Картой профилактических прививок пациента.

Пользовательский сценарий

Актор - врач общей практики

Сценарий: Оформить прием

Предварительное условие: У пациента есть медицинская карта.

Основной сценарий:

1.            Врач общей практики выбирает строку с записью пациента в расписании.

2.            Врач общей практики открывает электронную медицинскую карту пациента.

3.            Система отображает данные пациента и время записи.

4.            Врач сверяет личность пациента с данными в системе.

5.            Врач заполняет данные осмотра, основной диагноз, жалобы пациента, анамнез заболевания, данные общего осмотра, рекомендации.

6.            Врач сохраняет внесённые данные.

7.            Система проверяет все данные и сохраняет их.

8.            Система отображает сообщение пользователю о завершении приёма и готовности результатов обследования.

9.            Врач закрывает ЭМК пациента.

Исключение 1а. Если пациент без предварительной записи:

1. Врач нажимает кнопку «Принять без записи» на панели управления списком.

2. Система отображает форму «Человек: Поиск».

3. Врач вводит поисковый критерий в поля фильтра.

3. Врач нажимает кнопку «Найти».

4. Врач выбирает человека в списке найденных записей.

5. Система отображает Электронную медицинскую карту пациента.

Исключение 4а: Если данные пациента в системе отличаются от реальных.

1.            Врач исправляет данные пациента и повторно подтверждает верификацию.

2.            Система сохраняет новые данные и сохраняет сведения о верификации.

Исключение 7а: Не заполнены обязательные поля по приему пациента

1.            Система в результате проверки обнаруживает ошибки и не сохраняет данные.

2.            Система отображает сообщение об ошибках.

3.            Врач дозаполняет обязательные поля и повторно сохраняет данные.

4.            Система сохраняет данные.

Требования к системе

1.            ЭМК пациента должна быть доступна в интерфейсе из списка всех пациентов (меню «Пациенты») или из расписания (меню «Расписание»).

2.            Прием пациентов врачом возможен в случае, если текущий день у врача в расписании значится рабочим.

3.            Действие «Принять пациента» доступно только в случае, если пациент записан на текущий день.

4.            Действие «Принять без записи» доступно при отсутствии у пациента записи к врачу данного профиля на текущий день.

5.            В режиме работы «Прием пациента» реализован алгоритм ввода данных, которые получает врач при осмотре пациента. Прием пациента при обращении может регистрироваться как «Первичный прием» и «Повторный прием».
